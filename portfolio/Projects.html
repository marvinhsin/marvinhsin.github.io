PintOS: Uniprocessor Operating System

Pintos is an uniprocessor operating system for the x86 architecture. It supports multithreading, loading and running user programs, and a file system. Pintos could, theoretically, run on a regular IBM-compatible PC. But, we ran Pintos projects ​in Bochs2 and QEMU3 simulators simulator, that is, a program that simulates an x86 CPU and its peripheral devices accurately enough that unmodified operating systems and software can run under it. 

  • Built a uniprocessor operating system in C and Assembly Language supporting multithreading. 
  • Designed and implemented kernel modules including multithreads management, user programs management and virtual memory module. 
  • Supported 15+ syscalls, floating point operation and file system operations with strict priority scheduler.

Gitlet: Version Control System

Gitlet is a version-control system that mimics some of the basic features of the popular system Git. Ours is smaller and simpler, however, so we have named it Gitlet. The main functionality that Gitlet supports is:

  1. Saving the contents of entire directories of files. In Gitlet, this is called committing, and the saved contents themselves are called commits.
  2. Restoring a version of one or more files or entire commits. In Gitlet, this is called checking out those files or that commit.
  3. Viewing the history of your backups. In Gitlet, you view this history in something called the log.
  4. Maintaining related sequences of commits, called branches.
  5. Merging changes made in one branch into another.
  •  Built a version control system that simulates Git in Java from scratch to 1500+ lines of codes. 
  • Supported 13 essential git commands such as add, commit, status, checkout, log, branch, and merge.  
  • Designed customized method find to search versions with certain keyword over all committed messages.
End-to-End Encrypted File Sharing System

A handdraft workflow chart of the system. It is very sketchy, but memorable.

End-to-End Encrypted File Sharing System is a secure file sharing system. Imagine something similar to Dropbox, but secured with cryptography so that the server cannot view or tamper with your data.

  1. Authenticate with a username and password;
  2. Save files to the server;
  3. Load saved files from the server;
  4. Overwrite saved files on the server;
  5. Append to saved files on the server;
  6. Share saved files with other users; and
  7. Revoke access to previously shared files.
  • Built a secure file sharing system like Dropbox in Go with knowledge of cryptography.·      
  • Designed the user login system and secure file sharing process and implemented 8+ relevant client APIs.  
  • Applied efficient data structures to improve the file sharing, overwriting, appending, uploading and downloading processes to optimize performance efficiency and bandwidth usage.                              
Pacman: AI Agent in Python

The Pac-Man project is a semester-length long project. We are expected to apply an array of AI techniques to playing Pac-Man to accomodate different environments and conditions and find the best winning strategies. However, these projects don’t focus on building AI for video games. Instead, they teach AI concepts, such as informed state-space search, probabilistic inference, and reinforcement learning. These concepts underly real-world application areas such as natural language processing, computer vision, and robotics.

  1. Depth-first, breadth-first, uniform cost, and A* search algorithms. These algorithms are used to solve navigation and traveling salesman problems in the Pacman world.
  2. Multiagent SearchClassic Pacman is modeled as both an adversarial and a stochastic search problem. Students implement multiagent minimax and expectimax algorithms, as well as designing evaluation functions.
  3. Reinforcement Learning: Value Function, Q learning, and Approximate Q learning to help pacman and crawler agents learn rational policies.
  4. Bayes Nets and HMMs: Pacman uses probabilistic inference on Bayes Nets and the forward algorithm and particle sampling in a Hidden Markov Model to find ghosts given noisy readings of distances to them.
  5. Machine Learning: the perceptron algorithm, neural network, and recurrent nn models, and apply the models to several tasks including digit classification and language identification.
  •  Utilized 10+ search algorithms such as DFS, BFS to find optimal solutions to different search problems.    
  • Applied 5+ machine learning models such as Q-Learning, Neural Network to train the agent to score optimally given different environments. 
  •   Achieved 98% accuracy on classifications and 90% winning rate of games.
 Fault Tolerant MapReduce System in C

MapReduce is a programming model for scalable and highly parallelized data processing. It abstracts away the complexities of developing a fault tolerant distributed system by exposing a simple API where the user specifies two functions:

Map: produces a set of key/value pairs from the input data
Reduce: combines values corresponding to the same key

With these functions, tasks can be automatically parallelized and executed on a cluster. The structure is loosely based on a lab from MIT, The system design is similar to that outlined in the MapReduce paper.

  • Built a cloud distributed system that handles multiple map and reduce tasks in C with RPC. ·      
  • Implemented socket programming by TCP/IP to create a scalable and reliable system, allowing for efficient data transfer and communication between multiple users. 
  •   Implemented error handling and data validation techniques to ensure data integrity and prevent data loss.
